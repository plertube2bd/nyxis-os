# 🇺🇸 English Version

## Nyxis OS

Nyxis OS is an operating system being developed with a strong focus on user experience and interface consistency. By utilizing centralized APIs and unified design rules, it aims to provide a seamless and predictable UX across the entire system.

Rather than prioritizing legacy compatibility, Nyxis OS is designed to move toward modern technologies and experimental system concepts. The name “Nyxis” originates from *Nyx*, the Greek goddess of the night, combined with the suffix *-is* to give the impression of “a tendency toward something.”

Nyxis OS is being designed as an object-oriented operating system with scalability and extensibility as core principles.

---

## ❓ Why should I use Nyxis OS?

* Lightweight and user-oriented operating system
* Small and concise codebase
* Fully open-source
* Extreme abstraction layer (planned)
* Designed for learning and development

⚠️ Nyxis OS is an experimental hobby project.
Do NOT use it in production, security-critical, or automation-focused environments.

---

## 🔧 What about built-in utilities?

Good question.

Nyxis utilities are intentionally original and are not designed for strict UNIX or DOS compatibility. Instead, the shell is expected to abstract and bridge interactions between utilities and the operating system.

The naming convention for core utilities follows the format:

```text
__n_coreutils_*
```

# 🇰🇷 한국어 버전

## Nyxis OS

Nyxis OS는 강력한 사용자 UX를 취급하여 만들어지고 있는 운영체제입니다. 중앙 API와 규칙을 이용해 일관된 사용자 경험을 유도하며, 레거시 호환보단 신 기술을 향해 더 나아가려는 운영체제입니다.
Nyxis OS의 뜻은 밤의 여신을 뜻하는 nyx와, ~is의 무엇을 하는 경향이라는 느낌을 주기 위해 네이밍되었습니다.
Nyxis OS는 객체지향성 운영체제로 설계되고 있으며, 확장성을 위한 운영체제입니다.

## ❓ 왜 Nyxis OS를 사용해야 하나요?

## 🔧 기본 유틸리티는 무엇인가요?

좋은 질문입니다.

Nyxis의 유틸리티는 기본적으로 UNIX, DOS 호환이 아닌 독창적인 유틸리티들이며, 이는 쉘이 추상화 해야하는 대상입니다.
기본 유틸리티의 네이밍은 아래와 같아야합니다.
```text
__n_coreutils_*
```
